Data Layout


The ' /data ' URL give you access to get and set data on different object levels.

A brief breakdown of each level is as follows:

Sensor Level

The Sensor level is found at /data/sensor/ and is assumed to focus on the physical sensors themselves. There can be unique fields assigned to the sensor, but at a minimum, all sensors must contain:

  • ID
  • Type
  • Spot

Getting Data

To get sensor data from the API service, simply call a GET request to the respective URL for the sensor you're looking for.

GET " eel5632.tylersmith.us/data/sensor/YOUR_SENSOR_ID_HERE "

This can be done by simply entering the above URL in a browser window. This should return a JSON object with relevant sensor information, if the sensor exists. Otherwise, it will return a blank JSON object.

You can also access the sensor data from a browser in a more human-friendly format by using /data/sensor/view/YOUR_SENSOR_ID_HERE.

Please note that this returns an HTML webpage and should not be used for sending data to another service if just the JSON object is required.

Creating / Setting Data

To set data for a given sensor, simply call a POST request that will create a sensor if one doesn't already exist, and update the values with the provided information in a JSON object. To update a sensor, use:

POST " eel5632.tylersmith.us/data/sensor/YOUR_SENSOR_ID_HERE "

Be sure to pass up the API Authentication Key in the POST request as a component of the JSON object -- the API will scan the JSON data for the "key" keyword and authenticate with the provided ID and Key to the authentication service.

You do not need to provide all of the required fields if the sensor ID already exists -- the API will append whatever is currently existing to your updated data JSON object.

You can also change the spot that a sensor is assigned to by adding a different spot ID to the "spot" tag in the JSON object. I don't know why you would, but I added it, and that's pretty neat-o, as the kids say these days.
